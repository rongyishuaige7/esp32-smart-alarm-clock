# 安全、网络与隐私边界

## 适用范围

这是 ESP32 与 Flutter 的局域网教学原型，不是消费级闹钟、远程管理系统、安防设备、医疗设备或安全关键控制器。

## 网络边界

- 无 Wi-Fi 凭据时，ESP32 建立开放 AP `ESP32_Alarm_Config`，配网页面在 `http://192.168.4.1:81`；不要在此页面输入会复用到个人、学校或企业系统的密码；
- STA 成功连接后，REST API 使用**明文 HTTP** 端口 80；没有 TLS、身份认证、会话、CSRF 保护、速率限制或设备身份；
- REST 设置 `Access-Control-Allow-Origin: *`，只为了隔离可信局域网内的教学客户端互操作；
- Android 的 `usesCleartextTraffic="true"` 和 iOS 的 `NSAllowsLocalNetworking` 是本地 HTTP 兼容配置，绝不表示通信安全；
- 不要通过端口映射、反向代理、DDNS、校园网桥接、公共 Wi-Fi 或公网暴露该设备；
- 设备 API、传感器值和闹钟状态都不应被视为可信身份、安全监控、告警送达或物理执行确认。

## 硬件和音频边界

- ESP32 GPIO 为 3.3 V 逻辑。先核实每个模块的供电与电平，不要将 5 V 直接接到 GPIO；
- MAX98357A 与扬声器需要符合规格的供电和接线；GPIO 不直接驱动扬声器、电机、继电器、灯带或市电；
- 本仓库未包含来源不明的 WAV。只使用自己拥有再分发权的音频，且不要提交含个人语音、隐私信息、受限 TTS 或第三方铃声的文件；
- NVS 保存的 Wi-Fi 凭据只存在于设备本地。调试日志、截图、录屏和 Issue 中不得提交 SSID、密码、完整私网拓扑或设备标识。

## 报告问题

如果仓库启用了 GitHub 私密漏洞报告，请优先使用该入口；否则提交已脱敏的最小复现。不要附上 Wi-Fi 凭据、令牌、个人路径、真实局域网资产、录音或设备序列号。
